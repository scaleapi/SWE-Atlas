The scrollback history buffer in kitty implements a segmented circular buffer that allocates memory incrementally rather than reserving the full configured scrollback limit upfront. I began investigating this by examining the HistoryBuf implementation in history.c and running direct measurements against kitty's C data structures.

The fundamental storage unit is a segment of 2048 lines, defined by the SEGMENT_SIZE constant. Each segment holds CPUCell, GPUCell, and LineAttrs data for all cells in those lines. For an 80-column terminal, this translates to approximately 5 megabytes per segment.

```c
#define SEGMENT_SIZE 2048

static void
add_segment(HistoryBuf *self) {
    self->num_segments += 1;
    self->segments = realloc(self->segments, sizeof(HistoryBufSegment) * self->num_segments);
    HistoryBufSegment *s = self->segments + self->num_segments - 1;
    const size_t cpu_cells_size = self->xnum * SEGMENT_SIZE * sizeof(CPUCell);
    const size_t gpu_cells_size = self->xnum * SEGMENT_SIZE * sizeof(GPUCell);
    s->cpu_cells = calloc(1, cpu_cells_size + gpu_cells_size + SEGMENT_SIZE * sizeof(LineAttrs));
}
```

The interesting observation is that creating a buffer with capacity for 2048 lines immediately allocates one full segment (~5MB jump), but declaring capacity for 4096 or more lines does not allocate additional segments until lines are actually pushed. The segment_for function triggers allocation only when accessing lines beyond existing segments.

```c
static index_type
segment_for(HistoryBuf *self, index_type y) {
    index_type seg_num = y / SEGMENT_SIZE;
    while (UNLIKELY(seg_num >= self->num_segments && SEGMENT_SIZE * self->num_segments < self->ynum)) 
        add_segment(self);
    return seg_num;
}
```

The ~4.75 MB jumps align closely with the calculated 5.00 MB per segment, with the difference attributable to measurement granularity and allocator overhead. When the buffer reaches capacity at line 6144, the count stays at 6144 while new lines wrap around and overwrite the oldest data.

The scroll operation is effectively O(1) because it simply updates the scrolled_by counter rather than copying or moving any data. The screen_history_scroll function calculates the new position and calls dirty_scroll to mark the screen for re-rendering.

```c
bool
screen_history_scroll(Screen *self, int amt, bool upwards) {
    unsigned int new_scroll = MIN(self->scrolled_by + amt, self->historybuf->count);
    if (new_scroll != self->scrolled_by) {
        self->scrolled_by = new_scroll;
        dirty_scroll(self);
        return true;
    }
    return false;
}
```

When scrolling through history while new output arrives, the view position adjusts automatically. The test demonstrates this by scrolling to a middle position and then generating additional output:

The scrolled_by value remains stable at 4977 while history_line_added_count tracks how many lines were added. During rendering, screen_update_cell_data adjusts the view:

```c
if (self->scrolled_by) 
    self->scrolled_by = MIN(self->scrolled_by + history_line_added_count, self->historybuf->count);
```

The rendering rate is governed by repaint_delay in the main render loop. When scroll input arrives faster than this delay allows, frames coalesce naturally:

```c
static void
render(monotonic_t now, bool input_read) {
    static monotonic_t last_render_at = MONOTONIC_T_MIN;
    monotonic_t time_since_last_render = last_render_at == MONOTONIC_T_MIN ? OPT(repaint_delay) : now - last_render_at;
    if (!input_read && time_since_last_render < OPT(repaint_delay)) {
        set_maximum_wait(OPT(repaint_delay) - time_since_last_render);
        return;
    }
}
```

Signs of prioritization appear when typing during scroll. The input handler resets scrolled_by to zero, returning focus to the cursor position immediately:

```c
// Since we are typing, scroll to the bottom
if (self->scrolled_by != 0) {
    self->scrolled_by = 0;
    dirty_scroll(self);
}
```

The measurement scripts used for this analysis are included below. They can be run directly against a built kitty source tree:

```python
# /tmp/measure_historybuf.py (abbreviated)
from kitty.fast_data_types import HistoryBuf, LineBuf, Cursor

hb = HistoryBuf(10000, 80)
lb = LineBuf(1, 80)
cursor = Cursor()
lb.line(0).set_text('X' * 80, 0, 80, cursor)

for i in range(10000):
    hb.push(lb.line(0))
    if i % 2048 == 0:
        print(f"Lines: {i}, RSS: {get_process_memory_mb():.1f} MB")
```
