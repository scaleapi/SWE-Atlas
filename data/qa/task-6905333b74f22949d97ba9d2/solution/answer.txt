This demonstrates how the three organizational elements work together: Correspondent groups documents by sender/source (one-to-many), Document Type categorizes documents by purpose/format (one-to-many), and Tags provide flexible cross-cutting labels (many-to-many). All three support automatic assignment through rule-based matching (MATCH_ANY, MATCH_ALL, MATCH_LITERAL, MATCH_REGEX, MATCH_FUZZY) or ML-based classification (MATCH_AUTO with a trained neural network).

Based strictly on executed commands and observed outputs, paperless-ngx uses a single, consistent document flow. Documents enter via folder watcher, API, or email, and all enqueue the same `consume_file` task through django-q with Redis. The consumer runs pre-checks, parsing/OCR, thumbnail generation, date extraction, transactional storage, and signal-based enrichment (inbox tags → correspondent → document type → tags → log entry → search index). Required fields are populated at consumption time, optional organizational fields are populated by matching/classification, and path properties are derived at runtime from stored filenames and configuration. Tags provide many-to-many labeling while correspondents and document types provide one-to-many categorization.
