Then I searched for where padding gets added and found this in `scapy/arch/linux.py`:

def send(self, x):
    # type: (Packet) -> int
    try:
        return SuperSocket.send(self, x)
    except socket.error as msg:
        if msg.errno == 22 and len(x) < conf.min_pkt_size:
            padding = b"\x00" * (conf.min_pkt_size - len(x))
            if isinstance(x, Packet):
                return SuperSocket.send(self, x / Padding(load=padding))
            else:
                return SuperSocket.send(self, raw(x) + padding)
        raise


so padding only happens during send, not in memory. I also found in config.py:
min_pkt_size = 60

Then I looked at how scapy figures out what layer comes next after ethernet. Found this in packet.py:

def guess_payload_class(self, payload):
    for t in self.aliastypes:
        for fval, cls in t.payload_guess:
            try:
                if all(v == self.getfieldval(k)
                       for k, v in fval.items()):
                    return cls
            except AttributeError:
                pass
    return self.default_payload_class(payload)

def default_payload_class(self, payload):
    return conf.raw_layer


so if ethertype isn't bound, it just falls back to Raw. Then I found how padding gets extracted during dissection:

def dissect(self, s):
    s = self.pre_dissect(s)
    s = self.do_dissect(s)
    s = self.post_dissect(s)
    payl, pad = self.extract_padding(s)
    self.do_dissect_payload(payl)
    if pad and conf.padding:
        self.add_payload(conf.padding_layer(pad))


and in inet.py, IP uses its length field to figure out where padding starts:

def extract_padding(self, s):
    tmp_len = self.len - (self.ihl << 2)
    if tmp_len < 0:
        return s, b""
    return s[:tmp_len], s[tmp_len:]

So basically what I found is:

- scapy does NOT auto-pad packets in memory. a basic ether/ip/tcp is only 54 bytes which is below the 60 byte minimum.
- padding only gets added during `send()` in `scapy/arch/linux.py` when the kernel returns errno 22.
- when you dissect a padded packet with `Ether(raw_bytes)`, the IP layer's `extract_padding()` uses the length field to separate payload from padding, and if `conf.padding` is true it becomes a `Padding` layer.
- for unknown ethertypes like 0x9000 or 0x1234, scapy just falls back to `Raw` layer with no warnings or errors. it checks `payload_guess` bindings first, then calls `default_payload_class()` which returns `conf.raw_layer`.
