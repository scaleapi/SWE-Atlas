I started my investigation by creating a test script to observe Scapy's packet dissection at runtime. The core mechanism lives in the Packet class defined in scapy/packet.py, where the dissect method orchestrates the parsing of raw bytes into structured layer objects.

To understand how layers hand off to one another, I constructed a packet representing an HTTP GET request with Ethernet, IP, TCP, and raw payload layers, then converted it to raw bytes and dissected it back. I created the following test script:

$ cat > test_http_dissection.py << 'EOF'
#!/usr/bin/env python3
from scapy.all import *

http_payload = b"GET / HTTP/1.1\r\nHost: example.com\r\n\r\n"
pkt = Ether(dst="ff:ff:ff:ff:ff:ff", src="aa:bb:cc:dd:ee:ff", type=0x0800) / \
      IP(src="192.168.1.100", dst="93.184.216.34", proto=6) / \
      TCP(sport=54321, dport=80, flags="PA") / \
      Raw(load=http_payload)

raw_bytes = raw(pkt)
dissected = Ether(raw_bytes)

print("Packet layers:")
for i, layer in enumerate(dissected.layers()):
    print(f"  Layer {i}: {layer.__name__}")

print()
print(dissected.show2(dump=True))
EOF

$ python3 test_http_dissection.py

```

  Layer 2: TCP
  Layer 3: Raw

###[ Ethernet ]###

The dissection proceeds through a chain of method calls. When a Packet subclass receives raw bytes in its constructor, it calls self.dissect(_pkt) if the bytes are non-empty. The dissect method in scapy/packet.py coordinates the parsing by calling pre_dissect for preprocessing, do_dissect to parse fields from bytes, post_dissect for postprocessing, extract_padding to separate payload from padding, and finally do_dissect_payload to recursively dissect the payload.

This method first calls guess_payload_class to determine what type of packet should be created from the remaining bytes. The guess_payload_class method consults the payload_guess list, which contains tuples of field conditions and corresponding layer classes.

The bindings are created using bind_layers calls in the layer definition files. I verified this by searching the source:

$ grep -n "bind_layers(Ether, IP" scapy/layers/inet.py

Output:
```
1101:bind_layers(Ether, IP, type=2048)
```

This registers that when Ether.type equals 0x0800 (2048 decimal), the payload should be parsed as IP. The bind_layers function calls both bind_bottom_up to add entries to payload_guess for dissection and bind_top_down to set overload_fields for packet building.

When Scapy encounters an unrecognized payload, it falls back gracefully to the Raw layer. I tested this by creating a packet with custom binary data that matches no known protocol:

$ cat > test_unrecognized_payload.py << 'EOF'
#!/usr/bin/env python3
from scapy.all import *

custom_data = b"\xDE\xAD\xBE\xEF\x00\x01\x02\x03CUSTOM_PROTOCOL_DATA_HERE"
pkt_custom = Ether() / IP() / UDP(sport=9999, dport=9999) / Raw(load=custom_data)
dissected_custom = Ether(raw(pkt_custom))

# Get the final layer
final_layer = dissected_custom
while final_layer.payload and type(final_layer.payload).__name__ != 'NoPayload':
    final_layer = final_layer.payload

print(dissected_custom.getlayer(UDP).show2(dump=True))
print(f"Final layer type: {type(final_layer).__name__}")
print(f"Final layer class: {type(final_layer)}")
print(f"Is it Raw?: {isinstance(final_layer, Raw)}")
print(f"Raw.load length: {len(final_layer.load)} bytes")
EOF

When guess_payload_class finds no matching condition in payload_guess, it calls default_payload_class which returns conf.raw_layer, configured as the Raw class. This means any unrecognized data simply becomes a Raw layer containing the bytes in its load field, preserving the data without raising errors.

For the header trust test, I crafted a deliberately malformed packet where the Ethernet type field claims IPv6 (0x86DD) but the actual payload bytes are IPv4:

$ cat > test_header_trust.py << 'EOF'
#!/usr/bin/env python3
from scapy.all import *

ip4_pkt = IP(src="10.0.0.1", dst="10.0.0.2") / TCP(sport=1234, dport=80)
ip4_bytes = raw(ip4_pkt)

print(f"IPv4 packet bytes ({len(ip4_bytes)} bytes):")
print(f"First byte (version field): 0x{ip4_bytes[0]:02x} -> version = {ip4_bytes[0] >> 4}")

malicious_ether = Ether(dst="ff:ff:ff:ff:ff:ff", src="aa:bb:cc:dd:ee:ff", type=0x86DD)
malicious_raw = raw(malicious_ether) + ip4_bytes
dissected_malicious = Ether(malicious_raw)

print(f"\nEther.type value: 0x{dissected_malicious.type:04x}")
print(f"What Scapy thinks the payload is: {type(dissected_malicious.payload).__name__}")
print()
print(dissected_malicious.show2(dump=True))
EOF

$ python3 test_header_trust.py

Output:
```
IPv4 packet bytes (40 bytes):
First byte (version field): 0x45 -> version = 4

Ether.type value: 0x86dd
What Scapy thinks the payload is: IPv6

###[ Ethernet ]### 
  type      = IPv6
###[ IPv6 ]### 
     version   = 4
     tc        = 80
     fl        = 40
     plen      = 1
     nh        = Hop-by-Hop Option Header
     src       = 4006:66cd:a00:1:a00:2:4d2:50
     dst       = ::5002:2000:76be:0
```

Scapy blindly follows the Ethernet type field header. Even though the actual bytes start with 0x45 indicating IPv4, Scapy parses them as IPv6 because the Ethernet type said so. The resulting IPv6 layer shows version=4, which is clearly wrong for IPv6, but Scapy does not detect or report this inconsistency. The IPv4 bytes get reinterpreted through IPv6 field parsing, producing nonsensical addresses like 4006:66cd:a00:1:a00:2:4d2:50. This design choice prioritizes trusting protocol headers over heuristic detection, which aligns with how real network stacks behave but can lead to surprising results with malformed packets.

Scapy parses all the way down through the tunnel. The resulting packet object has 5 nested layers, and each can be accessed either by traversing the payload chain or using indexed access like dissected_gre.getlayer(IP, nb=2) to get the second IP layer.

I created another test to demonstrate the layer-by-layer traversal and bidirectional linking:

$ cat > test_layer_traversal.py << 'EOF'
#!/usr/bin/env python3
from scapy.all import *

inner_pkt = IP(src="172.16.0.1", dst="172.16.0.2", ttl=64) / ICMP(type=8, id=1234)
outer_pkt = Ether(type=0x0800) / IP(src="10.0.0.1", dst="10.0.0.2", proto=47) / \
            GRE(proto=0x0800) / inner_pkt
dissected_gre = Ether(raw(outer_pkt))

def traverse_layers(pkt, depth=0):
    indent = "  " * depth
    layer_name = type(pkt).__name__
    print(f"{indent}Depth {depth}: {layer_name}")
    if layer_name == "IP":
        print(f"{indent}  src={pkt.src}, dst={pkt.dst}")
        print(f"{indent}  proto={pkt.proto}")
    elif layer_name == "GRE":
        print(f"{indent}  proto={pkt.proto}")
    if pkt.payload and type(pkt.payload).__name__ != 'NoPayload':
        traverse_layers(pkt.payload, depth + 1)

print("Layer-by-layer traversal:")
traverse_layers(dissected_gre)

print(f"\nFirst IP layer (outer): src={dissected_gre.getlayer(IP, nb=1).src}, dst={dissected_gre.getlayer(IP, nb=1).dst}")
print(f"Second IP layer (inner): src={dissected_gre.getlayer(IP, nb=2).src}, dst={dissected_gre.getlayer(IP, nb=2).dst}")
icmp_layer = dissected_gre.getlayer(ICMP)
print(f"ICMP layer found: type={icmp_layer.type}, id={icmp_layer.id}")
EOF

$ python3 test_layer_traversal.py

Output:
```
Layer-by-layer traversal:
Depth 0: Ether
  Depth 1: IP
    src=10.0.0.1, dst=10.0.0.2
    proto=47
    Depth 2: GRE
      proto=2048
      Depth 3: IP
        src=172.16.0.1, dst=172.16.0.2
        proto=1
        Depth 4: ICMP

First IP layer (outer): src=10.0.0.1, dst=10.0.0.2
Second IP layer (inner): src=172.16.0.1, dst=172.16.0.2

The nested structure is maintained through the payload and underlayer attributes. Each layer's payload points to the next layer down, while underlayer points back up. This bidirectional linking allows navigation in both directions and enables methods like haslayer and getlayer to search the entire stack. The layers method returns a list of all layer classes in order, confirming that dissection successfully identified Ether, IP, GRE, IP, and ICMP.

The dissection path through the code follows a predictable pattern. When Ether receives raw bytes, it calls dissect, which parses the 14-byte Ethernet header using fields_desc, extracting dst, src, and type. The remaining bytes and the type value are passed to do_dissect_payload. The guess_payload_class method finds that type=2048 matches the binding for IP, so it creates an IP instance with the remaining bytes. IP then does the same: parses its header, sees proto=47, finds the GRE binding, creates a GRE instance. GRE parses its header, sees proto=2048, creates another IP instance. That inner IP sees proto=1, creates ICMP. ICMP parses its fields and has no payload guess for generic echo data, so dissection terminates with the packet fully parsed.
