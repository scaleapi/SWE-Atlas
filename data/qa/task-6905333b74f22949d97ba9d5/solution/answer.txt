The system tracks document processing state through multiple mechanisms:

1. Task Queue Status: The django_q_task table records whether each consumption task succeeded (success=1) or failed (success=0), allowing the system to identify failed ingestion attempts
2. Document Existence: Once successfully processed, documents exist in the documents_document table with a unique ID
3. File Removal: Successfully consumed files are deleted from the consumption directory (as shown in the log "Deleting file /app/src/../consume/complete_test.txt"), so only unprocessed files remain
4. Checksum Tracking: The checksum field in documents_document serves as the primary indicator that a document has been fully processed and stored

When a task fails (success=0), the file remains for retry unless deleted. When a task succeeds (success=1), the document record is created and the source file is removed, indicating processing is complete.

```python
    def pre_check_duplicate(self):
        with open(self.path, "rb") as f:
            checksum = hashlib.md5(f.read()).hexdigest()
        if Document.objects.filter(
            Q(checksum=checksum) | Q(archive_checksum=checksum),
        ).exists():
            if settings.CONSUMER_DELETE_DUPLICATES:
                os.unlink(self.path)
            self._fail(
                MESSAGE_DOCUMENT_ALREADY_EXISTS,
                f"Not consuming {self.filename}: It is a duplicate.",
```

This method calculates an MD5 checksum of the incoming file and queries the documents_document table for existing documents with matching checksum or archive_checksum values. If a match is found, it raises a ConsumerError, preventing further processing.

The duplicate check runs early in try_consume_file, before parsing or thumbnail generation:

```bash
grep -B 2 -A 2 "self.pre_check_duplicate" /app/src/documents/consumer.py
```

```python
        self.pre_check_file_exists()

```

This placement ensures duplicate files are rejected efficiently without wasting resources on parsing, OCR, or indexing. The checksum-based approach detects content-identical files regardless of filename differences.
