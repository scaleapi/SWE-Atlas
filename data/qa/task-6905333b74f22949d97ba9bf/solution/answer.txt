When I construct layers like `Ether()`, `IP(...)`, or `TCP(...)`, each call goes through `Packet.__init__` in `packet.py`. The constructor initializes `fields` and `payload`, calls `init_fields()` to populate `default_fields` from the class's `fields_desc`, then converts keyword arguments via each `Field`'s `any2i` method. At this point, `build()` hasn't been called fields like `ihl`, `len`, and `chksum` remain `None`.

When I stack layers with `pkt = eth / ip / tcp`, the `/` operator calls `Packet.__div__`, which clones both operands, calls `add_payload()`, and sets up parent/underlayer linkage. Still no bytes are emitted. The `show()` output reflects this internal `fields` state `ihl = None`, `len = None`, `chksum = None` for IP and `dataofs = None`, `chksum = None` for TCP.

Calling `raw(pkt)` triggers serialization. `raw()` calls `bytes(pkt)`, which invokes `Packet.__bytes__` > `build()`. The `build()` method calls `do_build()`, which runs `self_build()` to walk `fields_desc` and assemble headers via each field's `addfield()`, then recursively calls `do_build_payload()` for nested layers. After building, `post_build(pkt, pay)` runs for each layer. `IP.post_build` computes `ihl`, `len`, and `chksum`; `TCP.post_build` computes `dataofs` and `chksum` using the pseudo-header via `in4_chksum`. Only after these `post_build` overrides complete does `build()` return the final byte string with all `None` fields concretely encoded.

When I send with `sendp(pkt, iface=conf.iface, verbose=1)`, Scapy's `sendp` in `sendrecv.py` calls `_send`, which uses `resolve_iface()` to get a `NetworkInterface`, constructs a `SuperSocket` via `l2socket()`, and delegates to `__gen_send`. This loop calls `s.send(p)` for each packet, invoking `Packet.build()` if needed. Since destination `1.2.3.4` is unreachable, `conf.neighbor` fails to find a MAC address and logs warnings about using broadcast. With `verbose=1`, `__gen_send` prints "Sent 1 packets." after completion.

cd /app && python3 - << 'PY'

ip = IP(dst="1.2.3.4", src="5.6.7.8")

Since live sniffing didn't work here, I can emulate dissection by feeding `raw(pkt)` bytes back to `Ether(raw_bytes)`. When I call `Ether(raw_bytes)`, `Packet.__init__` sees non-empty `_pkt`, calls `dissect(_pkt)`, which runs `do_dissect` a loop through `fields_desc` calling each field's `getfield()`. Then `do_dissect_payload` calls `guess_payload_class()` to determine the next layer (IP from Ethernet's type field), and recursively instantiates it with `IP(s, _internal=1, _underlayer=self)`, continuing through TCP.

In my experiment, the original packet's `show()` displays `ihl = None`, `len = None`, `chksum = None` for IP and `dataofs = None`, `chksum = None` for TCP. After reparsing with `Ether(raw_bytes)`, these fields show concrete values: `ihl = 5`, `len = 40`, `chksum = 0x6abc` for IP, and `dataofs = 5`, `chksum = 0x4b5e` for TCP values extracted by `do_dissect` from bytes written by `post_build` during serialization.

In summary, the runtime trace for my Ether/IP/TCP packet works as follows: construction uses `Packet.__init__` to set logical fields without building bytes; stacking uses `Packet.__div__` and `add_payload` to wire up the layer tree; serialization for `raw(pkt)` flows through `Packet.__bytes__` > `Packet.build` > `do_build` > `self_build`, then IP and TCP `post_build` overrides compute headers and checksums; sending routes through `sendp` > `_send` > `__gen_send` > `SuperSocket.send` via `resolve_iface`; and dissection via `Ether(raw_bytes)` triggers `Packet.dissect` > `do_dissect` > `do_dissect_payload`, recursively instantiating IP and TCP with concrete field values from the raw frame.
