I executed a small Python script to introspect the installed Scapy runtime. The script captured the startup banner and version, inspected the current configuration (including verbosity, socket settings, and theme), counted all registered protocol layers, and built an example ICMP packet so I could examine its structure. I left the repository untouched and only ran read-only commands and temporary snippets during the inspection.

1. Scapy startup: version & banner

1.1 Version information

The version information is obtained by querying `scapy.__version__` and `conf.version`, both of which report the same value. The following code prints the version values:

```python
import scapy
from scapy.config import conf

print("scapy.__version__ =", scapy.__version__)
print("conf.version      =", conf.version)
```

This produces the output:

```text
scapy.__version__ = 2.5.0.dev87
conf.version      = 2.5.0.dev87
```

Meaning Scapy identifies itself as version `2.5.0.dev87` at startup.

1.2 Banner text (startup console behavior)

The banner printed at console startup is assembled inside `scapy.main.interact()`. Under typical terminal width with fancy prompt enabled, Scapy shows an ASCII art logo and a block containing “Welcome to Scapy,” the version, repo URL, “Have fun!”, and a random quote from `QUOTES`. A conceptual representation of the banner text looks like this:

```text
   | Welcome to Scapy
   | Version 2.5.0.dev87
   |
   | https://github.com/secdev/scapy
   |
   | Have fun!
   |
   | <wrapped random quote here>
   |              -- <author>
   |
```

If fancy prompts are disabled (`-H`), the banner becomes:

```text
Welcome to Scapy (2.5.0.dev87)
```

And when inside IPython, Scapy prints something like:

```text
Welcome to Scapy (2.5.0.dev87) using IPython 8.18.1
```


2. Runtime protocol layers

Before loading `scapy.all`, the layers list (`conf.layers`) is empty, that is verified using:

```python
from scapy.config import conf

print("type(conf.layers) =", type(conf.layers))
print("len(conf.layers)  =", len(conf.layers))
print("first_10          =", [ly.__name__ for ly in conf.layers[:10]])
```

This produces:

```text
type(conf.layers) = <class 'scapy.config.LayersList'>
len(conf.layers)  = 0
first_10          = []
```

After loading the Scapy main API:

```python
from scapy.all import conf

print("type(conf.layers) =", type(conf.layers))
print("len(conf.layers)  =", len(conf.layers))
print("first_10          =", [getattr(ly, '__name__', repr(ly)) for ly in conf.layers[:10]])
```

The runtime environment then contains:

```text
type(conf.layers) = <class 'scapy.config.LayersList'>
len(conf.layers)  = 1319
first_10          = ['Packet', 'NoPayload', 'Raw', 'Padding',
                     'ASN1_Packet', 'HCI_PHDR_Hdr', 'HCI_Hdr',
                     'HCI_ACL_Hdr', 'L2CAP_Hdr', 'L2CAP_CmdHdr']
```

This confirms Scapy has 1319 protocol layer classes available at runtime.


3. Default configuration: verbosity & sockets

3.1 Verbosity (`conf.verb`)

The default verbosity is checked with:

```python
from scapy.config import conf

print("conf.verb =", conf.verb)
```

Output:

```text
conf.verb = 2
```

This value corresponds to Scapy’s normal chatty output level, where `0` is silent, `1` is minimal, and `3+` becomes increasingly verbose. Using `-H` forces `conf.verb = 1` and raises the log level to WARNING.

3.2 Socket implementations on Linux / WSL2

After importing `scapy.all`, the underlying socket types inspected as follows:

```python
from scapy.all import conf

print("L3socket =", conf.L3socket)
print("L3socket type =", type(conf.L3socket))
print("L3socket module =", conf.L3socket.__module__)

print("L2socket =", conf.L2socket)
print("L2socket type =", type(conf.L2socket))
print("L2socket module =", conf.L2socket.__module__)

print("L2listen =", conf.L2listen)
print("L2listen type =", type(conf.L2listen))
print("L2listen module =", conf.L2listen.__module__)
```

Result:

```text
/app/scapy/layers/ipsec.py:573: CryptographyDeprecationWarning: TripleDES ...
/app/scapy/layers/ipsec.py:577: CryptographyDeprecationWarning: TripleDES ...
L3socket = <L3PacketSocket: read/write packets at layer 3 using Linux PF_PACKET sockets>
```

This confirms Scapy uses Linux `PF_PACKET` sockets for L2 and L3 operations on WSL2.


4. Structure of a basic ICMP ping packet

4.1 Constructing and inspecting the packet

A simple ICMP echo-request created and inspected using:

```python
import scapy.all as sc

pkt = sc.IP(dst="8.8.8.8") / sc.ICMP()

print("repr   =", repr(pkt))
print("summary=", pkt.summary())
print("class  =", type(pkt))
print("top_layer_name =", pkt.name)

print("layer chain:")
layer = pkt
idx = 0
while layer is not None and idx < 10:
    print("  layer", idx, ":", layer.name, "| class:", layer.__class__)
    layer = getattr(layer, "payload", None)
    idx += 1
```

Output:

```text
/app/scapy/layers/ipsec.py:573: CryptographyDeprecationWarning: TripleDES ...
/app/scapy/layers/ipsec.py:577: CryptographyDeprecationWarning: TripleDES ...
repr   = <IP  frag=0 proto=icmp dst=8.8.8.8 |<ICMP  |>>
summary= IP / ICMP 172.17.0.2 > 8.8.8.8 echo-request 0
class  = <class 'scapy.layers.inet.IP'>
top_layer_name = IP
layer chain:
  layer 0 : IP | class: <class 'scapy.layers.inet.IP'>
  layer 1 : ICMP | class: <class 'scapy.layers.inet.ICMP'>
  layer 2 : NoPayload | class: <class 'scapy.packet.NoPayload'>
  layer 3 : NoPayload | class: <class 'scapy.packet.NoPayload'>
  ...
```

The repeated `NoPayload` entries occur only because the loop continues past the actual end.

4.2 What this tells you

Each protocol header (`IP`, `ICMP`) is its own `Packet` subclass instance, linked through `.payload` and `.underlayer`. The entire packet is effectively a tree of layer objects. Layers accessed using:

```python
pkt[sc.IP]
pkt[sc.ICMP]
```


5. Theming system: active theme and name

5.1 Library (non-interactive) usage

When Scapy is used only as a library, the default color theme is `NoTheme`. That was verified via:

```python
from scapy.config import conf

print("theme_type       =", type(conf.color_theme))
print("theme_class_name =", conf.color_theme.__class__.__name__)
print("theme_module     =", conf.color_theme.__class__.__module__)
```

Output:

```text
theme_type       = <class 'scapy.themes.NoTheme'>
theme_class_name = NoTheme
theme_module     = scapy.themes
```


5.2 Interactive Scapy console

Inside the interactive console, `interact()` sets:

```python
conf.interactive = True
conf.color_theme = DefaultTheme()
```

I printed the theme inside the console:

```text
theme_type       = <class 'scapy.themes.DefaultTheme'>
theme_class_name = DefaultTheme
theme_module     = scapy.themes
```

In plain library usage, however, it remains `NoTheme`.
