When Scapy parses a packet from raw bytes or builds one from scratch, the decision to rebuild from fields versus returning cached bytes happens through the interplay of three packet attributes and the build chain in packet.py. The caching mechanism exists as a performance optimization that avoids redundant serialization, but it creates counterintuitive behavior when modifying nested structures.

The three controlling attributes on every Packet instance are raw_packet_cache which stores the original serialized bytes for that layer only, raw_packet_cache_fields which stores a snapshot of mutable field values at parse time, and explicit which indicates whether the packet was parsed from wire bytes or constructed programmatically.

During packet dissection, the do_dissect method populates the cache immediately after parsing each field. The raw bytes consumed by this layer are stored, and for any field that is mutable, holds packets, or is a list, a snapshot of its value is captured for later comparison.

```python
# From scapy/packet.py (lines 1002-1021)
def do_dissect(self, s):
    _raw = s
    self.raw_packet_cache_fields = {}
    for f in self.fields_desc:
        if not s:
            break
        s, fval = f.getfield(self, s)
        if isinstance(f, ConditionalField) and fval is None:
            continue
        if (f.islist or f.holds_packets or f.ismutable) and fval is not None:
            self.raw_packet_cache_fields[f.name] = \
                self._raw_packet_cache_field_value(f, fval, copy=True)
        self.fields[f.name] = fval
    self.raw_packet_cache = _raw[:-len(s)] if s else _raw
    self.explicit = 1
    return s
```

The critical detail lies in what gets cached for PacketListField and similar fields that contain nested packets. The _raw_packet_cache_field_value method explicitly avoids copying entire packet structures for performance reasons, instead only capturing the fields dictionary of each nested packet.

```python
# From scapy/packet.py (lines 648-662)
def _raw_packet_cache_field_value(self, fld, val, copy=False):
    _cpy = lambda x: fld.do_copy(x) if copy else x
    if fld.holds_packets:
        # avoid copying whole packets (perf: #GH3894)
        if fld.islist:
            return [
                _cpy(x.fields) for x in val
            ]
        else:
            return _cpy(val.fields)
    elif fld.islist or fld.ismutable:
        return _cpy(val)
    return None
```

This design choice means that for a PacketListField containing items, only each item's fields dictionary is snapshotted. The payload of those nested packets, any nested PacketField or PacketLenField inside them, and any deeper nesting is not tracked at all.

When building a packet by calling bytes() or raw(), the do_build method orchestrates the process. For packets that were not parsed from wire bytes, the iteration mechanism creates a clone with fixed volatile values. For parsed packets, the existing fields are used directly.

```python
# From scapy/packet.py (lines 724-740)
def do_build(self):
    if not self.explicit:
        self = next(iter(self))
    pkt = self.self_build()
    for t in self.post_transforms:
        pkt = t(pkt)
    pay = self.do_build_payload()
    if self.raw_packet_cache is None:
        return self.post_build(pkt, pay)
    else:
        return pkt + pay
```

The decision point at line 737 determines whether post_build runs. If raw_packet_cache is still set, the cached bytes are concatenated with the payload bytes directly, completely bypassing post_build. This means any checksum or length calculations defined in post_build will not execute.

The self_build method performs the actual cache validation. It compares current field values against the snapshot stored in raw_packet_cache_fields. If any mutable field has changed, the cache is invalidated and the packet is rebuilt from field values.

```python
# From scapy/packet.py (lines 678-713)
def self_build(self):
    if self.raw_packet_cache is not None and \
            self.raw_packet_cache_fields is not None:
        for fname, fval in self.raw_packet_cache_fields.items():
            fld, val = self.getfield_and_val(fname)
            if self._raw_packet_cache_field_value(fld, val) != fval:
                self.raw_packet_cache = None
                self.raw_packet_cache_fields = None
                self.wirelen = None
                break
        if self.raw_packet_cache is not None:
            return self.raw_packet_cache
    p = b""
    for f in self.fields_desc:
        val = self.getfieldval(f.name)
        if isinstance(val, RawVal):
            p += bytes(val)
        else:
            p = f.addfield(self, p, val)
    return p
```

Direct field assignment triggers cache invalidation through the setfieldval method. When you assign to a field on a packet, the cache is cleared for that specific layer.

```python
# From scapy/packet.py (lines 472-487)
def setfieldval(self, attr, val):
    if attr in self.default_fields:
        fld = self.get_field(attr)
        if fld is None:
            any2i = lambda x, y: y
        else:
            any2i = fld.any2i
        self.fields[attr] = val if isinstance(val, RawVal) else \
            any2i(self, val)
        self.explicit = 0
        self.raw_packet_cache = None
        self.raw_packet_cache_fields = None
        self.wirelen = None
```

Regular payload chain modifications behave differently because the payload is rebuilt independently:


This works because do_build calls do_build_payload which recursively builds the payload chain. Each layer's cache is checked independently. When the payload's field is modified, its cache is cleared, so it rebuilds. The outer layer's cache only covers its own bytes, not the payload bytes.

The show() method versus bytes() discrepancy occurs because show() reads from the fields attribute directly using getfieldval, which returns the modified in-memory value. But bytes() returns the cached serialization for layers where the cache is still valid.

The show2 method works by building the packet to bytes and then parsing those bytes into a completely new packet object for display:

```python
# From scapy/packet.py (lines 1473-1486)
def show2(self, dump=False, indent=3, lvl="", label_lvl=""):
    return self.__class__(raw(self)).show(dump, indent, lvl, label_lvl)
```

For freshly constructed packets where raw_packet_cache is None, show2 triggers a full build including post_build which calculates checksums. But the original packet object is never modified. Each show2 call produces a new temporary packet from the current bytes.

The iteration mechanism in __iter__ handles volatile fields by fixing their values before building. When explicit is False and no cache exists, the iterator collects all VolatileValue instances and creates a clone with those values fixed:

```python
# From scapy/packet.py (lines 1124-1163)
def __iter__(self):
    def loop(todo, done, self=self):
        if todo:
            eltname = todo.pop()
            elt = self.getfieldval(eltname)
            if not isinstance(elt, Gen):
                if self.get_field(eltname).islist:
                    elt = SetGen([elt])
                else:
                    elt = SetGen(elt)
            for e in elt:
                done[eltname] = e
                for x in loop(todo[:], done):
                    yield x
        else:
            if isinstance(self.payload, NoPayload):
                payloads = SetGen([None])
            else:
                payloads = self.payload
            for payl in payloads:
                done2 = done.copy()
                for k in done2:
                    if isinstance(done2[k], VolatileValue):
                        done2[k] = done2[k]._fix()
                pkt = self.clone_with(payload=payl, **done2)
                yield pkt

    if self.explicit or self.raw_packet_cache is not None:
        todo = []
        done = self.fields
    else:
        todo = [k for (k, v) in itertools.chain(self.default_fields.items(),
                                                self.overloaded_fields.items())
                if isinstance(v, VolatileValue)] + list(self.fields)
        done = {}
    return loop(todo, done)
```

The clone_with method creates a new packet instance that inherits the raw_packet_cache from the original. This preserves caching for parsed packets even through the iteration process:

```python
# From scapy/packet.py (lines 1103-1122)
def clone_with(self, payload=None, **kargs):
    pkt = self.__class__()
    pkt.explicit = 1
    pkt.fields = kargs
    pkt.default_fields = self.copy_fields_dict(self.default_fields)
    pkt.overloaded_fields = self.overloaded_fields.copy()
    pkt.raw_packet_cache = self.raw_packet_cache
    pkt.raw_packet_cache_fields = self.copy_fields_dict(
        self.raw_packet_cache_fields
    )
    if payload is not None:
        pkt.add_payload(payload)
    return pkt
```

The clear_cache method provides explicit cache invalidation that recurses through nested packets and payloads:

```python
# From scapy/packet.py (lines 664-676)
def clear_cache(self):
    """Clear the raw packet cache for the field and all its subfields"""
    self.raw_packet_cache = None
    for fname, fval in self.fields.items():
        fld = self.get_field(fname)
        if fld.holds_packets:
            if isinstance(fval, Packet):
                fval.clear_cache()
            elif isinstance(fval, list):
                for fsubval in fval:
                    fsubval.clear_cache()
    self.payload.clear_cache()
```

Using clear_cache resolves the nested modification problem:

For the checksum-before-length problem in freshly built packets, the order of operations in post_build is critical. The payload is fully built before post_build is called, so length can always be computed. But if the implementation calculates checksum before updating length, the checksum will be wrong:

```python
# test_show2_twice.py - WRONG order
class ProblematicPacket(Packet):
    def post_build(self, pkt, pay):
        # BUG: Calculate checksum BEFORE updating length
        if self.checksum is None:
            full = pkt + pay
            cksum = sum(full) & 0xFF
            pkt = pkt[:2] + bytes([cksum]) + pkt[3:]
        
        # Length updated AFTER checksum (wrong!)
        if self.length is None:
            total_len = len(pkt) + len(pay)
            pkt = pkt[0:1] + bytes([total_len]) + pkt[2:]
        return pkt + pay
```

```
======================================================================
```

The checksum is 57 instead of 64 because when post_build calculates the checksum, the length byte is still 0 (the default), not 7. The correct implementation updates length first:

```python
# CORRECT order
class CorrectPacket(Packet):
    def post_build(self, pkt, pay):
        # Update length FIRST
        if self.length is None:
            total_len = len(pkt) + len(pay)
            pkt = pkt[0:1] + bytes([total_len]) + pkt[2:]
        
        # Then calculate checksum with correct length
        if self.checksum is None:
            full = pkt + pay
            cksum = sum(full) & 0xFF
            pkt = pkt[:2] + bytes([cksum]) + pkt[3:]
        return pkt + pay
```

```
======================================================================

Second bytes(): 010740deadbeef
```


Both show2() calls display checksum=57 because the underlying post_build bug persists. The original packet's checksum field remains None throughout. If show2() twice appeared to fix the checksum in the user's scenario, it may involve volatile fields that fix on iteration, or a different caching interaction specific to their protocol implementation.

The fundamental design tension is between performance and mutation semantics. Deep comparison of entire nested packet trees on every build would be expensive, so Scapy opts for shallow comparison of immediate fields only. This makes caching fast but creates the documented limitation where deeply nested modifications require explicit cache invalidation via clear_cache().
