I started by building the sftpgo binary. I ran:

```bash
cd /app && go build -o sftpgo_binary . 2>&1
```

then i set up a test directory and created ssh keys:

```bash
mkdir -p /tmp/sftpgo_test/users/testuser && mkdir -p /tmp/sftpgo_test/backups
ssh-keygen -t ed25519 -f /tmp/sftpgo_test/id_ed25519 -N '' -q
ssh-keygen -t rsa -b 2048 -f /tmp/sftpgo_test/id_rsa -N '' -q
```

then i initialized the sqlite database with the schema files:

```bash
cd /tmp/sftpgo_test && sqlite3 sftpgo.db < /app/sql/sqlite/20190828.sql && sqlite3 sftpgo.db < /app/sql/sqlite/20191112.sql && sqlite3 sftpgo.db < /app/sql/sqlite/20191230.sql && sqlite3 sftpgo.db < /app/sql/sqlite/20200116.sql
```

i created a config file at `/tmp/sftpgo_test/sftpgo.json` with quota tracking enabled and started the server:

```bash
cd /app && ./sftpgo_binary serve --config-dir /tmp/sftpgo_test --log-file-path /tmp/sftpgo_test/sftpgo.log 2>&1 &
```

then i deleted a file to test the size quota:

```bash
sshpass -p 'testpass123' sftp -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null -P 2222 testuser@127.0.0.1 <<EOF
rm /testfile2.txt
bye
EOF
```

now i was at 2 files, 4019 bytes. then i tried uploading the 6000-byte file which would exceed the 5000 byte limit:

```bash
sshpass -p 'testpass123' sftp -v -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null -P 2222 testuser@127.0.0.1 <<EOF 2>&1
put /tmp/hugeoverquota.bin /hugeoverquota.bin
bye
EOF
```

surprisingly it worked! the upload succeeded because the quota check only looks at current usage, not the incoming file size. the server logs showed:

