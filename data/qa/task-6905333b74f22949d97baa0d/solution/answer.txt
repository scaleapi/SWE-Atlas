then I looked at the source code in pkg/engine/ahocorasick/ahocorasickcore.go to see how keywords are mapped to detectors
```go
func NewAhoCorasickCore(allDetectors []detectors.Detector, opts ...CoreOption) *Core {
    keywordsToDetectors := make(map[string][]DetectorKey)
    detectorsByKey := make(map[DetectorKey]detectors.Detector, len(allDetectors))
    var keywords []string
    for _, d := range allDetectors {
        key := CreateDetectorKey(d)
        detectorsByKey[key] = d
        for _, kw := range d.Keywords() {
            kwLower := strings.ToLower(kw)
            keywords = append(keywords, kwLower)
            keywordsToDetectors[kwLower] = append(keywordsToDetectors[kwLower], key)
        }
    }
```

next I looked at the deduplication cache key format in pkg/engine/engine.go
```go
        key := fmt.Sprintf("%s%s%s%+v", result.DetectorType.String(), result.Raw, result.RawV2, result.SourceMetadata)
        if val, ok := e.dedupeCache.Get(key); ok && (val != result.DecoderType ||
            result.SourceType == sourcespb.SourceType_SOURCE_TYPE_POSTMAN) {
            continue
        }
        e.dedupeCache.Add(key, result.DecoderType)
```
the key is made of detector type, raw secret, rawv2, and source metadata. The deduplication logic works as follows: when the same dedupe key is found with a different decoder type, the duplicate is skipped. To demonstrate this, I scanned separate files containing the same secret in plaintext and base64 formats
```bash
./trufflehog filesystem /tmp/trufflehog_test/secrets.txt /tmp/trufflehog_test/encoded_secrets.txt --no-verification
```
and I got two results
```text
Found unverified result 🐷🔑❓
Detector Type: Twilio
Decoder Type: PLAIN
File: /tmp/trufflehog_test/secrets.txt
Line: 1

Found unverified result 🐷🔑❓
Detector Type: Twilio
Decoder Type: BASE64
File: /tmp/trufflehog_test/encoded_secrets.txt
Line: 1
```
both appear because they have different SourceMetadata (different files). The dedupe key includes the file path, so secrets found in different files always produce distinct keys and both get reported. If the exact same secret were found by both decoders at the identical file and line, only the first one would be reported due to deduplication.
