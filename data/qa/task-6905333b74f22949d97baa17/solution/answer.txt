Here's why Case B is accepted despite SPF `-all` (the precise mechanism in this commit). The decision is made by the SPF check module `apply_spf` during end-of-`DATA` processing (body time): it calls `relyOnDMARC`, and if that returns true, it logs `deferring action due to a DMARC policy` and then deliberately clears `Reject` and `Quarantine` even if SPF failed:

```330:365:/app/internal/check/spf/spf.go
func (s *state) CheckBody(ctx context.Context, header textproto.Header, body buffer.Buffer) module.CheckResult {
	if s.c.enforceEarly {
		// Already applied in CheckConnection.
		return module.CheckResult{}
	}

	defer trace.StartRegion(ctx, "apply_spf/CheckBody").End()

	res, ok := <-s.spfFetch
	if !ok {
		return module.CheckResult{
			Reject: true,
			Reason: exterrors.WithTemporary(
				exterrors.WithFields(errors.New("panic recovered"), map[string]interface{}{
					"check":    "spf",
					"smtp_msg": "Internal error during policy check",
				}),
				true,
			),
		}
	}
	if s.relyOnDMARC(ctx, header) {
		if res.res != spf.Pass || res.err != nil {
			s.log.Printf("deferring action due to a DMARC policy")
		} else {
			s.log.Debugf("deferring action due to a DMARC policy")
		}

		checkRes := s.spfResult(res.res, res.err)
		checkRes.Quarantine = false
		checkRes.Reject = false
		return checkRes
	}

	return s.spfResult(res.res, res.err)
}
```

The deferral trigger condition is: a DMARC record is found for the relevant policy domain and the effective DMARC policy is not `none` (and for subdomains, `sp=` is honored when the record was found on the organizational domain). That logic is here:

```212:235:/app/internal/check/spf/spf.go
func (s *state) relyOnDMARC(ctx context.Context, hdr textproto.Header) bool {
	fromDomain, err := maddydmarc.ExtractFromDomain(hdr)
	if err != nil {
		s.log.Error("DMARC domains extract", err)
		return false
	}

	policyDomain, record, err := maddydmarc.FetchRecord(ctx, dns.DefaultResolver(), fromDomain)
	if err != nil {
		s.log.Error("DMARC fetch", err, "from_domain", fromDomain)
		return false
	}
	if record == nil {
		return false
	}

	policy := record.Policy
	// TODO: Is it ok to use EqualFold for subdomain check?
	if !dns.Equal(policyDomain, fromDomain) && record.SubdomainPolicy != "" {
		policy = record.SubdomainPolicy
	}

	return policy != dmarc.PolicyNone
}
```

And the DMARC lookup behavior that makes Case A special (try `_dmarc.<fromDomain>` first, then fall back to the organizational domain) is implemented here:

```21:52:/app/internal/dmarc/evaluate.go
func FetchRecord(ctx context.Context, r Resolver, fromDomain string) (policyDomain string, rec *Record, err error) {
	policyDomain = fromDomain

	// 1. Lookup using From Domain.
	txts, err := r.LookupTXT(ctx, "_dmarc."+fromDomain)
	if err != nil {
		dnsErr, ok := err.(*net.DNSError)
		if !ok || !dnsErr.IsNotFound {
			return "", nil, err
		}
	}
	if len(txts) == 0 {
		// No records or 'no such host', try orgDomain.
		orgDomain, err := publicsuffix.EffectiveTLDPlusOne(fromDomain)
		if err != nil {
			return "", nil, err
		}

		policyDomain = orgDomain

		txts, err = r.LookupTXT(ctx, "_dmarc."+orgDomain)
		if err != nil {
			dnsErr, ok := err.(*net.DNSError)
			if !ok || !dnsErr.IsNotFound {
				return "", nil, err
			}
		}
		// Still nothing? Bail out.
		if len(txts) == 0 {
			return "", nil, nil
		}
	}
```

So, with `subdomain.maddy-dmarc.com`, DMARC is found only via the org-domain fallback on `maddy-dmarc.com`. Because that record has `sp=none`, the effective subdomain policy becomes `none`, so `relyOnDMARC` returns false and SPF is enforced (Case A rejects). With `maddy-dmarc.com` itself, the policy is `p=reject` (not `none`), so `relyOnDMARC` returns true and `apply_spf` defers enforcement (Case B accepts). With `maddy-no-dmarc.com`, no DMARC record exists at all, so there is no deferral and SPF is enforced (Case C rejects).

Once `apply_spf` returns a `CheckResult` where `Reason != nil` but both `Reject` and `Quarantine` are false (the deferral path), the message pipeline logs `no check action` and continues:
