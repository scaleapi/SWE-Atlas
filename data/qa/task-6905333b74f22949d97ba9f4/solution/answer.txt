I built kitty from source by running the setup script:

`cd /app && python3 setup.py build`

The build completed successfully, compiling the GLFW wayland module and the Go-based kitten binary. To identify all compiled extension modules produced, I ran:

`find /app -name "*.so" -type f`

The output showed four extension modules:

```
/app/kitty/fast_data_types.so
/app/kitty/glfw-x11.so
/app/kitty/glfw-wayland.so
/app/kittens/transfer/rsync.so
```

I then executed the full test suite using:

`cd /app && python3 setup.py test`

The test output began with environment information followed by individual test results:


The suite ran 145 Python tests with 138 passing, 2 errors related to zsh unicode handling, and 5 skipped due to environmental constraints.

To trace which extension modules actually get loaded during test execution, I instrumented the import mechanism and observed:

```
C Extension modules loaded:
  kitty.fast_data_types: /app/kitty/fast_data_types.so
  kittens.transfer.rsync: /app/kittens/transfer/rsync.so
```

The GLFW modules were not imported as Python extensions. The `test_glfw_modules` test verifies their existence via filesystem checks, and `test_utf_8_strndup` loads them through ctypes to test specific C functions.

To understand how failures cascade when extensions are unavailable, I first tested removing `fast_data_types.so`:

`mv kitty/fast_data_types.so kitty/fast_data_types.so.bak && python3 -c "from kitty_tests import *"`

The output showed:

```
Traceback (most recent call last):
  File "<string>", line 1, in <module>
  File "/app/kitty_tests/__init__.py", line 21, in <module>
    from kitty.config import finalize_keys, finalize_mouse_mappings
  File "/app/kitty/config.py", line 10, in <module>
    from .conf.utils import BadLine, parse_config_base
  File "/app/kitty/conf/utils.py", line 27, in <module>
    from ..fast_data_types import Color
ModuleNotFoundError: No module named 'kitty.fast_data_types'
```

The import chain revealed that `kitty_tests/__init__.py` imports from `kitty.config`, which imports from `kitty.conf.utils`, which imports from `kitty.fast_data_types`. Since the test framework's BaseTest class cannot be defined without this extension, all 145 tests fail when it's missing. I restored the file after testing.

I then tested removing `rsync.so`:

`mv kittens/transfer/rsync.so kittens/transfer/rsync.so.bak && python3 -c "from kitty_tests.main import find_all_tests; find_all_tests()"`

The output showed:

```
Traceback (most recent call last):
  File "<string>", line 1, in <module>
  File "/app/kitty_tests/main.py", line 64, in find_all_tests
    m = importlib.import_module(package + '.' + x.partition('.')[0])
  File "/app/kitty_tests/file_transmission.py", line 13, in <module>
    from kittens.transfer.rsync import Differ, Hasher, Patcher, parse_ftc
ModuleNotFoundError: No module named 'kittens.transfer.rsync'
```

The `find_all_tests()` function iterates through all test modules and imports them. When it encounters `file_transmission.py`, which has a module-level import from `kittens.transfer.rsync`, test discovery crashes entirely. Zero tests execute because the suite cannot enumerate available tests. I restored the file after testing.

The dependency structure revealed by the test output shows that `kitty_tests.parser` and `kitty_tests.screen` require `fast_data_types` for Screen and Cursor classes. The `kitty_tests.datatypes` module needs it for Color, LineBuf, HistoryBuf, and width calculations. The `kitty_tests.graphics` module uses it for PNG loading and shared memory operations. The `kitty_tests.fonts` module depends on it for rendering functions. The `kitty_tests.file_transmission` module imports directly from `rsync` at module level.

Both `fast_data_types.so` and `rsync.so` are critical for test execution, without either one, zero tests run because test discovery fails before any test can be enumerated. The GLFW modules are optional since they're accessed via ctypes rather than Python imports, affecting less than 2% of tests when missing. No repository files were modified during this analysis.
