I traced kitty's startup shaping and fallback behavior by turning on its verbose font diagnostics and running the initialization path end-to-end, then capturing both the emitted logs and the runtime values that are in effect before any text is rendered.

To understand the initialization flow, I first examined the relevant source code. In `/app/kitty/fonts/render.py`, the `prerender_function` at lines 364-376 handles the initial sprite generation with parameters for cell dimensions, baseline, underline/strikethrough positions and thicknesses, cursor thicknesses, and DPI values. The C code in `/app/kitty/fonts.c` at lines 1450-1472 shows the sprite upload order - a blank cell is sent first, then the prerender callback is invoked to generate decoration sprites. The `sprite_map_set_limits` function in `/app/kitty/shaders.c` at lines 1223-1228 confirms that the first parameter sets `max_texture_size` and the second sets `max_array_texture_layers`.

Looking at how kitty handles complex Unicode, I found that text shaping is powered by HarfBuzz - each font face maintains an `hb_font_t` handle created during initialization (visible in `/app/kitty/freetype.c` at lines 226-228 where `hb_ft_font_create` sets up the shaping engine). This is what enables proper ligature rendering and complex script support. The `disable_ligatures` setting in `/app/kitty/options/types.py` at line 515 defaults to 0 (meaning `DISABLE_LIGATURES_NEVER` per the enum in `/app/kitty/data-types.h` line 68), so ligatures are enabled out of the box. For bidirectional text, the `force_ltr` option at line 525 defaults to `False`, which means kitty respects the natural right-to-left direction of Arabic and Hebrew text rather than forcing everything left-to-right. Combining diacritics are handled through the `cc_idx[3]` array in the CPUCell structure (`/app/kitty/data-types.h` line 226), which can store up to three combining marks per base character - the `is_combining_char` function in `/app/kitty/unicode-data.c` identifies these characters during text processing. This means kitty uses a decomposition approach for combining marks: rather than using precomposed characters, the base character is stored in the `ch` field while combining marks are stored separately in the `cc_idx[3]` array, allowing up to three diacritics per cell to be rendered on top of the base glyph during the shaping pass.

import sys
sys.path.insert(0, '/app')

from kitty.options.types import defaults
from kitty.options.utils import parse_font_spec
import kitty.fonts.render as render
from kitty.fast_data_types import (
    set_options,
    sprite_map_set_limits,
    set_send_sprite_to_gpu,
    create_test_font_group,
    current_fonts,
    get_fallback_font,
)

opts = defaults._replace(font_family=parse_font_spec('monospace'))
set_options(opts, False, True, True)

orig_prerender = render.prerender_function
def logging_prerender(*args, **kwargs):
    cw, ch, baseline, ul_pos, ul_thk, strike_pos, strike_thk, cursor_beam, cursor_ul, dpi_x, dpi_y = args
    print(f"PRERENDER metrics cw={cw} ch={ch} baseline={baseline} ul_pos={ul_pos} ul_thk={ul_thk} strike_pos={strike_pos} strike_thk={strike_thk} cursor_beam={cursor_beam} cursor_ul={cursor_ul} dpi=({dpi_x},{dpi_y})")
    return orig_prerender(*args, **kwargs)
render.prerender_function = logging_prerender

sprites = {}
atlas_initialized = False
def send_to_gpu(x, y, z, data):
    global atlas_initialized
    if not atlas_initialized:
        print("Atlas initialized: texture layers allocated, ready to receive sprites")
        atlas_initialized = True
    sprites[(x, y, z)] = data
    print(f"SPRITE upload to atlas cell coords x={x} y={y} z={z} bytes={len(data)}")

sprite_map_set_limits(8192, 256)
set_send_sprite_to_gpu(send_to_gpu)

try:
    render.set_font_family(opts)
    cell_width, cell_height = create_test_font_group(opts.font_size, 96.0, 96.0)
    print("HarfBuzz font handles: initialized and ready for text shaping")
    print(f"CELL metrics width={cell_width} height={cell_height}")
    render.dump_font_debug()
    print("SPRITE atlas slots used:", sorted(sprites.keys()))
finally:
    set_send_sprite_to_gpu(None)
    render.prerender_function = orig_prerender
    set_options(None)
EOF
