The caching system uses a tree-structured WeakMap dependency chain for cache lookups. Looking at the source code implementation in packages/tree-select/src/index.ts:
```
let cache = new WeakMap();

const leafCache: Map< string, Result > = dependents.reduce( insertDependentKey, cache );

const key = getCacheKey( ...args );
if ( leafCache.has( key ) ) {
    return leafCache.get( key ) as Result;
}

const value = selector( dependents, ...( args as SArgs ) );
leafCache.set( key, value );
return value;
```

The default cache key is generated by args.join() as shown in packages/tree-select/src/index.ts:
```
const defaultGetCacheKey: GenerateCacheKey< unknown[] > = ( ...args: unknown[] ): string =>
    args.join();
```

A cache hit occurs when the same dependent objects by reference identity are returned by the dependency getter and the cache key string matches an existing entry via leafCache.has( key ).

```
test( 'should bust the cache when clearCache() method is called', () => {
    const reduxState = {
        posts: {
            [ post1.id ]: post1,
            [ post2.id ]: post2,
            [ post3.id ]: post3,
        },
    };

    const firstResult = getSitePosts( reduxState, 'site1' );
    const memoizedResult = getSitePosts( reduxState, 'site1' );

    // Repeated call returns identical object
    expect( memoizedResult ).toBe( firstResult );

    getSitePosts.clearCache();
    const afterClearResult = getSitePosts( reduxState, 2916284 );

    // Is forced to compute a new result after clearCache()
    expect( afterClearResult ).not.toBe( firstResult );
} );
```

The test passing proves that at runtime the .clearCache() method works. The assertion expect( memoizedResult ).toBe( firstResult ) passed confirming caching before clear. The assertion expect( afterClearResult ).not.toBe( firstResult ) passed confirming cache was cleared.

The .clearCache() method is implemented in packages/tree-select/src/index.ts as:
```
cachedSelector.clearCache = () => {
    // WeakMap doesn't have `clear` method, so we need to recreate it
    cache = new WeakMap();
};
```

The test passing proves that at runtime for each primitive in [ true, 1, 'a', false, '', 0 ] calling the selector throws an error. The error comes from the insertDependentKey function in packages/tree-select/src/index.ts:
```
if ( key != null && Object( key ) !== key ) {
    throw new TypeError( 'key must be an object, `null`, or `undefined`' );
}
```

The error is TypeError: key must be an object, `null`, or `undefined` because WeakMap requires object keys.

The test that passed is in packages/tree-select/test/index.js "accepts a getCacheKey option that enables object arguments":
```
test( 'accepts a getCacheKey option that enables object arguments', () => {
    const reduxState = {
        posts: {
            [ post1.id ]: post1,
            [ post2.id ]: post2,
            [ post3.id ]: post3,
        },
    };

    const memoizedSelector = treeSelect(
        ( state ) => [ state.posts ],
        ( [ posts ], query ) => Object.values( posts ).filter( ( p ) => p.siteId === query.siteId ),
        { getCacheKey: ( query ) => `key:${ query.siteId }` }
    );

    // The arguments are objects, they are not identical, but generated keys are.
    // Therefore, the second call returns the memoized result.
    const firstResult = memoizedSelector( reduxState, { siteId: 'site1', foo: 'bar' } );
    const secondResult = memoizedSelector( reduxState, { siteId: 'site1', foo: 'baz' } );
    expect( firstResult ).toEqual( [ post1, post2 ] );
    expect( firstResult ).toBe( secondResult );
} );
```

The test passing proves that at runtime even though two calls used different object arguments { foo: 'bar' } versus { foo: 'baz' }, the custom getCacheKey extracted only siteId producing same cache key 'key:site1' for both. The assertion expect( firstResult ).toBe( secondResult ) passed confirming cache hit. Without custom getCacheKey, object arguments throw in development mode as shown by test "should throw an error in development when given object arguments" where:
```
expect( () => getSitePosts( state, {} ) ).toThrow();
expect( () => getSitePosts( state, [] ) ).toThrow();
```
